## 2. Why This Topic Matters

A roadmap can look productive while still having weak foundations. A checkpoint helps me avoid rushing forward without understanding the skills I already practiced. It also helps me turn daily learning into a stronger GitHub portfolio.

## 3. Core Concepts

Foundation review means checking whether the basics are clear enough to build on.

Skill auditing means listing what I can do confidently and what still needs practice.

Project review means checking if each project has a clear goal, readable code or notes, useful documentation, and a professional structure.

ML core preparation means getting ready for statistics, SQL, model evaluation, pipelines, cross-validation, and portfolio-level projects.

## 6. Common Mistakes

- Moving to advanced models before understanding data preparation.
- Treating every small script as a portfolio project.
- Ignoring documentation quality.
- Not reviewing weak areas before starting a harder phase.
- Overclaiming skill level instead of showing honest progress.

## 7. Interview-Style Questions

1. Why is EDA important before machine learning?
   It helps understand data quality, patterns, missing values, outliers, and possible feature relationships.
2. Why is train/test split important?
   It checks how a model performs on unseen data.
3. What is data leakage?
   It happens when information from outside the training process incorrectly helps the model.
4. Why should projects include limitations?
   Limitations make the project more honest and professional.
5. Why should a beginner review completed work?
   Review helps identify weak foundations before moving into harder topics.
